원칙(Principles)

  1. 모바일 생산(Mobile Production): 핸드폰에서 쓰고, 바로 퍼블리시.
  2. 자동화(Automation): 반복은 스크립트·에이전트(Agents)에게.

깃허브 페이지(Pages) 배포 스니펫

# 1) 새 레포 생성 후 메인 브랜치 푸시
git init && git add -A && git commit -m "init (mobile-first writing os)"
git branch -M main
git remote add origin <YOUR_REPO_URL>
git push -u origin main

# 2) GitHub Pages 설정: Settings → Pages → Source: GitHub Actions
# 3) .github/workflows/deploy.yml 추가
name: pages
on:
  push: { branches: [ main ] }
permissions: { contents: read, pages: write, id-token: write }
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: upload
        uses: actions/upload-pages-artifact@v3
        with: { path: "." }
  deploy:
    needs: build
    runs-on: ubuntu-latest
    permissions: { pages: write, id-token: write }
    steps:
      - uses: actions/deploy-pages@v4

Philosophy · 간단명료

“했던 거 또 하지 말자.” (Do Not Repeat Yourself)

LLM은 개발자(Developer)의 도구이자, 작가(Author)의 붓. 텍스트의 끝이 코드라면, 글쓰기는 곧 앱(App)이다.

  • Draft → Automate → Publish를 생활화.
  • 핸드폰에서 뼈대(Skeleton), PC에서 대량 처리(Batch).
  • 공개·비공개 라우팅으로 기억력 100% 달성.
